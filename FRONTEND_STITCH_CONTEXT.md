# SANKALP HACKATHON: Frontend UI Capsule (Google Stitch Specification)

This document is a high-fidelity design prompt, layout specification, and API binding contract tailored specifically for **Google Stitch** (Google Labs' AI UI generator) to generate a premium, high-impact React + Tailwind CSS dashboard for the **Artisan Women's Market**.

---

## 🎨 Design Language & Aesthetics

*   **Atmosphere**: Minimalistic, warm, clean, and highly accessible. Designed specifically for rural users to easily understand and learn without overwhelming visual clutter.
*   **Color Palette (Clean & Accessible)**:
    *   `Background`: Pure white (`#FFFFFF`) or very soft warm off-white (`#FCFBF9`) to ensure maximum readability and a neat, clean slate.
    *   `Primary (Heritage Orange)`: HSL `24 85% 50%` (`#E05A10`) - used sparingly for primary call-to-actions and important highlights.
    *   `Secondary (Deep Indigo)`: HSL `226 70% 20%` (`#1E1B4B`) - used for high-contrast, easily readable text.
*   **Typography**: Inter or Roboto (large, legible fonts with high contrast).
*   **Focus / Spotlight Onboarding Feature (Interactive Tutorial)**: Implement a "Focus Mode" tutorial system for basic app features. When introducing a feature to the user, a dark translucent overlay should blur the entire background (`backdrop-blur-sm bg-black/40`), leaving a single, bright cutout highlighting the exact button or element they need to click to guide their attention.

## 🧩 Visual App Layout (The 4 Core Hubs)

Stitch should generate a **Single Page Application Dashboard** with a responsive side-navigation menu:

```
+----------------------------------------------------------------------------------+
|  [Sidebar Navigation]        |  [Header: Village Node Sync Status (Green Dot)]   |
|  * Dashboard / Overview      +---------------------------------------------------+
|  * AI Voice & Storefront     |                                                   |
|  * AI Pricing & Demand       |                 Active Workspace                  |
|  * P2P Logistics & Routing   |                                                   |
|  * SHECAN Trust Ledger & QR  |                                                   |
+----------------------------------------------------------------------------------+
```

### 1. Unified Overview (The Rural Intelligence Center)
*   **Hero Cards (Metrics)**:
    *   **Trust Score**: Circular progress ring showing `9.8/10` (backed by the immutable ledger).
    *   **Villages Synced**: Local mesh indicator showing "3/3 Nodes Active".
    *   **Delivery Route Savings**: Card showing `~34% fuel savings` generated by P2P route clustering.
    *   **Active Listings**: Total count of AI-generated products.

---

### 2. AI Voice & Storefront Generator
*   **Goal**: Allow non-literate rural women to record voice + upload product images and auto-generate multilingual storefront cards.
*   **UI Components**:
    *   **Voice Recorder Widget**: An interactive microphone button with a pulse animation (`animate-ping`) when active, accompanied by a language selection dropdown (`Tamil`, `Hindi`, `Telugu`, `Kannada`, `Bengali`).
    *   **Image Dropzone / Camera Capture**: A file-drop field that simulates a scan laser line when an image is loaded, returning detected product categories using computer vision.
    *   **Generate Button**: Primary Orange CTA with a glowing border.
    *   **Output Cards**: High-fidelity product card previews showing title, pricing, and descriptions generated in multiple languages.
*   **API Binding**:
    *   `POST http://127.0.0.1:8000/voice/roundtrip`
        *   *Request*: `{ "audio_path": "string", "language_code": "ta" }`
        *   *Response*: `{ "transcription": "...", "translated_text": "...", "tts_audio_path": "..." }`
    *   `POST http://127.0.0.1:8000/products/detect`
        *   *Request*: `{ "image_path": "string" }`
        *   *Response*: `{ "detected_objects": [...], "confidence_score": 0.94 }`
    *   `POST http://127.0.0.1:8000/storefront/generate`
        *   *Request*: `{ "image_context": "string", "voice_transcript": "string", "language": "ta" }`
        *   *Response*: `{ "title": "...", "description": "...", "tags": [...], "storefront_markup": "..." }`

---

### 3. AI Fair Pricing & Demand Forecaster
*   **Goal**: Demystify fair trade value, compare direct artisan payout vs greedy middleman markups, and show peak seasonal demand windows.
*   **UI Components**:
    *   **Interactive Input Form**: Slider controls for:
        *   *Material Cost (INR)*
        *   *Labor Hours Spent*
        *   *Skill Multiplier (e.g. Master Weaver, Apprentice)*
    *   **Interactive Pricing Band Chart**: A comparative bar/gauge visualizer showcasing:
        *   *Recommended Price* (Middle)
        *   *Minimum Floor Price* (Fair Trade bottom)
        *   *Market Markup Comparison* (To emphasize how much more they earn directly vs through brokers).
    *   **Demand Trends Grid**: A calendar or timeline layout flagging Indian festivals (e.g., Diwali, Pongal) with glowing badge tags indicating "+45% Projected Demand Spike".
*   **API Binding**:
    *   `POST http://127.0.0.1:8000/pricing/predict`
        *   *Request*: `{ "material_cost": 250.0, "labor_hours": 12, "skill_level": "intermediate", "demand_factor": 1.2 }`
        *   *Response*: `{ "base_price": 850.0, "demand_adjusted_price": 935.0, "fair_price_band": [800.0, 1100.0], "recommended_price": 950.0 }`
    *   `POST http://127.0.0.1:8000/demand/forecast`
        *   *Request*: `{ "product_category": "textiles", "current_month": "October" }`
        *   *Response*: `{ "forecasted_demand": "high", "festival_uplift": 0.45, "primary_drivers": ["Diwali Shopping", "Winter Textiles"] }`

---

### 4. P2P Logistics & Network Router
*   **Goal**: Cluster nearby deliveries into shared transport routes and visualize network routing paths between rural locations.
*   **UI Components**:
    *   **Network Status Panel**: Interactive list of federated nodes (`Udupi`, `Mangalore`, `Karkala`, etc.) showing latency and local inventory counts.
    *   **Delivery Route Map Block**: An elegant mockup of an geographic canvas or real Map Frame showing connected dots representing paths.
    *   **Interactive Controls**: Dropdowns to pick `Source Hub` and `Destination Hub` with a "Compute Shared Route" button that returns exact mileage and waypoint lists.
    *   **Cluster Optimizer Card**: Slider to adjust target clustering (e.g., `Cluster nearby artisans into 4 batches`), instantly rendering localized groups.
*   **API Binding**:
    *   `GET http://127.0.0.1:8000/logistics/analytics`
        *   *Response*: `{ "total_artisans_mapped": 42, "optimized_route_savings_percent": 34.2, "carbon_reduction_kg": 112.5 }`
    *   `GET http://127.0.0.1:8000/logistics/clusters?n_clusters=4`
        *   *Response*: `{ "clusters": { "0": ["Udupi", "Malpe"], "1": ["Mangalore", "Ullal"] } }`
    *   `GET http://127.0.0.1:8000/logistics/route?source=Udupi&target=Ujire`
        *   *Response*: `{ "route": ["Udupi", "Karkala", "Ujire"], "total_distance_km": 72.4, "estimated_hours": 1.8 }`

---

### 5. SHECAN Trust Ledger & QR Passport
*   **Goal**: Prove the authenticity and origin of the craft to consumers via QR codes and demonstrate absolute data ownership via immutable transaction blocks.
*   **UI Components**:
    *   **Blockchain Transaction Log**: A scrolling, matrix-like green-and-gray ledger showing block cards:
        *   `Block #24 [VALID] - SHA-256: 0a9f...b3`
        *   *Data: Artisan Devika logged consent for craft origin details.*
    *   **Digital Passport Scanner / Previewer**:
        *   An image container displaying a beautifully styled **QR Product Passport**.
        *   Scanning or clicking this card pulls up the **Artisan Origin Story & Ledger History Timeline** (e.g., "Grown in Karkala -> Spun in Udupi -> Hand-woven by Devika").
    *   **Spam Detection Sandbox**: A demo button to "Submit 5 Fast Ratings" that triggers a warning dialog showing that the automated trust engine blocked the IP address and flagged it as `Flagged Fake: YES`.
*   **API Binding**:
    *   `POST http://127.0.0.1:8000/ledger/transaction`
        *   *Request*: `{ "product_id": "string", "artisan_id": "string", "transaction_type": "creation", "data_payload": {}, "client_ip": "192.168.1.1" }`
        *   *Response*: `{ "block_index": 5, "timestamp": "...", "hash": "...", "flagged_fake": false, "trust_score": 10.0 }`
    *   `GET http://127.0.0.1:8000/ledger/history/{product_id}`
        *   *Response*: `{ "product_id": "...", "chain": [...], "chain_is_valid": true }`
    *   `GET http://127.0.0.1:8000/ledger/passport/{product_id}`
        *   *Response*: `{ "product_id": "...", "qr_code_file_path": "...", "explorer_url": "..." }`
    *   `POST http://127.0.0.1:8000/ledger/sync`
        *   *Response*: `{ "status": "synced", "blocks_uploaded": 12 }`
